# 数据跨境合规审核法律智能体开发任务拆解

版本：V0.1  
日期：2026-05-31  
范围：MVP / 试点版

## 1. 交付目标

MVP 试点版需要交付一个可演示、可试点、可接入首批法律数据的系统，覆盖以下闭环：

1. 企业注册登录。
2. 创建数据合规审查任务。
3. 选择审查国家和场景。
4. 完成 5 轮智能体问答。
5. 保存问答和结构化字段。
6. 检索中国、越南、新加坡法律知识库。
7. 生成风险项、问题清单和整改建议。
8. 生成页面版报告和 PDF 报告。
9. 企业查看和下载历史报告。
10. 后台管理企业、任务、法律资料和报告。

## 2. 里程碑总览

| 阶段 | 周期 | 目标 | 主要产出 |
|---|---:|---|---|
| M1 需求与原型确认 | 1 周 | 锁定 MVP 范围和页面流程 | 原型、问答脚本、报告模板、数据库草案 |
| M2 基础系统开发 | 2-3 周 | 跑通企业端和后台基础流程 | 登录、任务、问答页、基础后台、核心表 |
| M3 知识库与智能体 | 2-3 周 | 接入法律资料并生成风险分析 | 清洗切片、检索、槽位抽取、风险项 |
| M4 报告与安全 | 1-2 周 | 生成正式报告并补齐权限 | 页面报告、PDF、权限、日志 |
| M5 测试与试点 | 1-2 周 | 面向 30 家以内企业试点 | 测试报告、修复清单、上线包 |

预计总周期：6-10 周。

## 3. M1：需求与原型确认

目标：把需求文档转成可开发、可验收的产品定义。

### 3.1 产品与流程

任务：

1. 确认 MVP 支持国家：中国、越南、新加坡。
2. 确认审查类型：境内数据合规审查、数据出海审查。
3. 确认企业端页面流程。
4. 确认管理后台页面流程。
5. 确认隐私承诺和免责声明文案。
6. 确认报告是否需要品牌样式。

验收：

1. 形成页面清单。
2. 形成用户流程图。
3. 形成待确认问题清单。

### 3.2 5 轮问答脚本

任务：

1. 设计第 1 轮企业基本信息问题。
2. 设计第 2 轮数据类型问题。
3. 设计第 3 轮数据流向问题。
4. 设计第 4 轮合规措施问题。
5. 设计第 5 轮补充确认问题。
6. 设计缺失字段追问规则。

验收：

1. 每轮问题都有目标槽位。
2. 每轮问题都能接受自然语言回答。
3. 高风险或缺失信息有追问策略。

### 3.3 报告模板

任务：

1. 设计页面报告结构。
2. 设计 PDF 报告结构。
3. 设计风险项展示格式。
4. 设计问题清单展示格式。
5. 设计法律依据引用格式。
6. 固定免责声明位置。

验收：

1. 报告包含企业信息、审查范围、风险等级、风险项、问题清单、整改建议、法律依据、免责声明。
2. 报告表达为初步审查，不构成正式法律意见。

### 3.4 数据库设计

任务：

1. 确认业务表。
2. 确认法律资料表。
3. 确认报告表。
4. 确认权限和日志表。
5. 确认字段命名和枚举值。

验收：

1. 输出 ER 草案。
2. 输出建表 SQL 草案。
3. 输出关键索引设计。

## 4. M2：基础系统开发

目标：跑通“企业创建任务 - 完成问答 - 保存记录”的基础闭环。

### 4.1 项目脚手架

任务：

1. 初始化前端项目。
2. 初始化后端项目。
3. 配置本地开发环境。
4. 配置数据库连接。
5. 配置环境变量。
6. 配置代码格式化和基础检查。

验收：

1. 前端可本地启动。
2. 后端可本地启动。
3. 数据库迁移可执行。

### 4.2 认证与企业信息

任务：

1. 注册接口。
2. 登录接口。
3. 当前用户接口。
4. 企业信息创建和更新。
5. 角色字段和基础权限。
6. 密码哈希存储。

验收：

1. 企业用户可以注册登录。
2. 登录后可以查看本企业信息。
3. 未登录用户不能访问受保护接口。

### 4.3 审查任务

任务：

1. 新建审查任务接口。
2. 审查任务列表接口。
3. 审查任务详情接口。
4. 审查类型枚举。
5. 国家选择枚举。
6. 任务状态流转。

验收：

1. 企业可以创建境内审查任务。
2. 企业可以创建中国 + 越南/新加坡出海审查任务。
3. 企业只能查看本企业任务。

### 4.4 隐私承诺确认

任务：

1. 隐私承诺展示组件。
2. 任务级确认记录。
3. 未确认时禁止进入问答。
4. 使用限制文案展示。

验收：

1. 用户必须确认后才能开始问答。
2. 确认记录可追溯。

### 4.5 问答页面

任务：

1. 聊天式问答界面。
2. 当前轮次展示。
3. 用户回答输入。
4. 消息保存接口。
5. 智能体问题生成接口。
6. 轮次结束状态。

验收：

1. 系统可以完成 5 轮问答。
2. 每轮消息保存到 audit_messages。
3. 页面刷新后能恢复问答记录。

### 4.6 管理后台基础版

任务：

1. 后台登录或管理员入口。
2. 企业列表。
3. 任务列表。
4. 任务详情。
5. 问答记录查看。

验收：

1. 平台管理员可以查看企业和任务。
2. 后台不允许普通企业用户访问。

## 5. M3：知识库与智能体

目标：将法律资料加工成可检索知识库，并接入智能体分析链路。

### 5.1 法律数据导入

任务：

1. 支持导入法律数据包。
2. 过滤 __MACOSX、.DS_Store、._* 文件。
3. 解析 JSON / CSV / NDJSON 元数据。
4. 重写本机绝对路径为相对路径或对象存储路径。
5. 写入 import_batches。
6. 写入 legal_documents。

验收：

1. 可以导入中国、越南、新加坡法律资料元数据。
2. 每条法规保留官方来源 URL。
3. 导入失败有错误记录。

### 5.2 HTML 清洗

任务：

1. 读取 content.txt。
2. 识别 HTML 内容。
3. 移除 script、style、nav、footer、重复导航。
4. 提取标题、章节、条文和正文。
5. 输出 clean_text。
6. 保存清洗质量指标。

验收：

1. 清洗后正文无明显网页导航噪音。
2. 每条法规都有可读纯文本。
3. 清洗结果可追溯到原始文件。

### 5.3 条文切片

任务：

1. 中国法规按“第 X 条”切片。
2. 新加坡法规按 Part / Section / Regulation 切片。
3. 越南法规按 “Điều” 切片。
4. 保留章、节、条上下文。
5. 写入 legal_clauses。
6. 记录切片失败项。

验收：

1. 每个切片关联原法规。
2. 每个切片保留国家、条文编号、来源 URL。
3. 抽检切片可读、边界合理。

### 5.4 标签体系

任务：

1. 建立基础标签表。
2. 支持标签绑定到法规和条文。
3. 对数据出境、个人信息、敏感个人信息、重要数据、数据本地化、告知同意、安全措施等做标签补全。
4. 管理后台支持查看和编辑标签。

验收：

1. 法律条文可以按标签过滤。
2. 标签修改后检索结果可更新。

### 5.5 检索服务

任务：

1. 国家过滤。
2. 领域过滤。
3. 标签过滤。
4. 全文检索。
5. 向量检索或预留向量检索接口。
6. 检索结果重排序。
7. 写入 retrieval_records。

验收：

1. 中国任务只检索中国库。
2. 中国 + 新加坡任务检索中国和新加坡库。
3. 中国 + 越南任务检索中国和越南库。
4. 检索结果可追溯到法规和条文。

### 5.6 槽位抽取

任务：

1. 定义核心槽位 schema。
2. 从用户回答中抽取企业名称、行业、业务场景、数据类型、跨境传输、接收方、授权、安全措施等字段。
3. 记录置信度。
4. 保存到 audit_slots。
5. 对低置信度字段进入追问。

验收：

1. 每轮回答后能抽取对应槽位。
2. 缺失字段能被识别。
3. 槽位可在后台查看。

### 5.7 风险项与问题清单

任务：

1. 实现基础规则引擎。
2. 识别个人信息跨境传输风险。
3. 识别敏感数据/重要数据风险。
4. 识别境外接收方缺失问题。
5. 识别授权、告知、协议、安全措施缺失问题。
6. 生成 risk_items。
7. 生成 question_items。

验收：

1. 风险项包含标题、等级、事实、依据、说明、建议。
2. 问题清单包含缺失信息、补充材料、律师判断事项。
3. 高风险和信息不足事项会提示人工复核。

## 9. 建议团队分工

最小团队：

1. 产品/项目负责人：需求确认、验收、试点反馈。
2. 前端工程师：企业端、后台、报告页面。
3. 后端工程师：API、数据库、权限、报告服务。
4. AI / RAG 工程师：知识库清洗、切片、检索、智能体编排。
5. 法务专家：法规确认、标签口径、报告抽检。
6. 测试工程师：流程、安全、报告、知识库质量测试。

若团队较小，后端可兼任 PDF 和后台，AI/RAG 工程师优先保障知识库质量。

## 10. 数据接入后的第一批任务

等法律数据提供后，优先执行以下任务：

1. 确认数据包目录结构。
2. 统计国家、领域、法规数量。
3. 检查编码、文件大小、HTML 噪音。
4. 检查是否包含官方 URL 和生效状态。
5. 建立导入批次。
6. 清洗 content.txt。
7. 按国家切条文。
8. 补齐标签。
9. 导入 legal_documents 和 legal_clauses。
10. 建立检索索引。
11. 用 10 个样例问题做召回测试。
12. 输出知识库质量报告。

## 11. 主要待确认问题

1. 是否需要平台管理员审核报告后再发给企业？
2. 法律资料是否由法务团队确认有效版本？
3. PDF 是否需要固定品牌样式？
4. 是否需要中英文报告？
5. 企业数据保留周期是多久？
6. 是否需要企业主动删除数据功能？
7. MVP 是否接入企业微信或第三方身份认证？
8. 税务和企业落地资料是否进入 MVP 报告正文，还是仅作为背景资料？
9. 是否需要律师端工作台？
10. 是否需要法规更新提醒机制？
